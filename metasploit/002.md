The focus here is not on completing a CTF, but on understanding the process step by step: from scanning and analysis to selecting and using the appropriate exploit.

From the results, we can clearly see that the target machine is vulnerable to **MS17-010 (EternalBlue)**, a critical remote code execution (RCE) vulnerability affecting SMBv1. This confirms that the SMB service on port 445 is exploitable, and that out next step woould be selecting the appropriate exploit to gain access to the system.

To exploit this vulnerability, we select the exploit `exploit/windows/smb/ms17_010_eternalblue` after launching `msfconsole`, and configure the required parameters:
- `RHOSTS` for the target machine
- `LHOST` for our local machine, and a suitable
- `PAYLOAD` (by default `windows/x64/meterpreter/reverse_tcp`)

Once everything is set, we run the exploit. If successful, the payload will be delivered to the target and establish a reverse connection back to us, resulting in a **meterpreter session** that allows us to interact with and control the compromised system.

## 🧠 Takeaways

- Scanning with `nmap` helps identify open ports and services, which guides the search for vulnerabilities.
- Using `--script vuln` can quickly highlight known vulnerabilities during enumeration.
- Gaining a **meterpreter session** allows more powerful interaction with the target system.
- Upgrading a shell improves stability and provides additional post-exploitation features.
- Privilege escalation (e.g. `getsystem`) is important to gain full control over the system.
- Migrating to a stable process helps maintain a reliable session.
- Post-exploitation involves navigating the file system and understanding key locations within the OS.
